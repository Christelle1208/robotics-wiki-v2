# Model: MolmoAct2 — Action Reasoning Models for Real-World Deployment

**Category:** Vision-Language-Action model with explicit action reasoning; real-world robot deployment

## What It Is

MolmoAct2 extends [[algo-molmo2]] (Molmo2) with **action reasoning** — the ability to not just describe or ground objects, but to generate explicit reasoning about *what actions to take* and then produce robot-executable action commands.

The key innovation over standard VLAs (like OpenVLA): rather than directly mapping observation → action tokens, MolmoAct2 first generates an **explicit reasoning chain** about the current situation, then produces actions conditioned on that reasoning. This chain-of-thought for manipulation is what the "Act2" name reflects — action via reasoning.

---

## How It Works

### From VLM to VLA via Action Reasoning

Standard VLA pipeline (e.g., OpenVLA):
```
[Image + Language instruction] → VLA → [Action tokens]
```

MolmoAct2 pipeline:
```
[Image + Language instruction] 
→ MolmoAct2 
→ [Reasoning text: "The red cup is near the edge. I should approach from the left to avoid knocking other objects."]
→ [Action tokens: joint position deltas]
```

The **reasoning text** is generated autoregressively before the action tokens, using Molmo2's language model. The action tokens then attend to the full context — including the generated reasoning — before being decoded.

### Architecture

**Base:** Molmo2 VLM (visual encoder + language model)

**Action head:**
- Additional vocabulary tokens for robot actions (similar to OpenVLA's discretization approach)
- The language model generates reasoning text first, then switches to generating action tokens
- A special token `[ACT]` marks the transition from language reasoning to action generation

**Training:**
- Fine-tuned on robot demonstration data annotated with reasoning traces
- Reasoning traces either written by humans or generated by a larger model (distillation) and then verified
- Action supervision: standard behavior cloning loss on the action tokens
- Language supervision: next-token prediction on the reasoning text

### Reasoning-Action Joint Training

The model is trained jointly on:
1. **Reasoning prediction loss:** Cross-entropy on reasoning tokens (maximize probability of the reasoning trace)
2. **Action prediction loss:** Cross-entropy / MSE on action tokens (maximize probability of the demonstrated action)

The reasoning trace conditions the action prediction, so the model learns to use explicit reasoning as a mechanism to improve action quality.

## Why Action Reasoning Helps

### Handling Ambiguity

Many manipulation scenarios are ambiguous from the observation alone:
- "Pick up the bottle" — which bottle? (reasoning resolves: "The instruction says 'the blue bottle'; I see one blue and one red, so I target the blue one")
- "Move it to the shelf" — which shelf level? (reasoning: "The shelf on the left is closer and within reach range")

Explicit reasoning forces the model to commit to an interpretation before acting, reducing action ambiguity.

### Out-of-Distribution Generalization

When the robot encounters a scenario not in its training data, the reasoning mechanism can:
- Describe what it sees
- Identify the most similar trained scenario
- Adapt the action strategy accordingly

This is analogous to chain-of-thought improving LLM generalization on novel math problems — explicit intermediate steps allow the model to generalize compositionally.

### Debuggability

The explicit reasoning trace is human-readable. When the robot fails, the operator can read the reasoning to understand *why* — "it thought the bottle was on the left shelf" — and correct the model's understanding, rather than treating the policy as a black box.

## Real-World Deployment Focus

MolmoAct2 specifically targets the challenges of real-world deployment (as opposed to simulation-only evaluation):

**Visual robustness:** The Molmo2 backbone, trained on diverse internet images, provides better robustness to:
- Variable lighting conditions
- Cluttered backgrounds
- Unfamiliar object appearances

**Language grounding:** Molmo2's grounding capability allows MolmoAct2 to precisely localize manipulation targets described in language, even for novel object types.

**Failure recovery:** The reasoning trace allows the robot to detect when a previous action failed ("I attempted to grasp but the gripper didn't close properly — I should retry with adjusted approach angle") and adapt.

## Evaluation

### Tasks
MolmoAct2 is evaluated on diverse real-world manipulation tasks:
- **Pick-and-place** with language-specified objects in cluttered scenes
- **Articulated object manipulation** (opening drawers, doors, containers)
- **Precision assembly** tasks requiring careful spatial reasoning
- **Multi-step sequences** requiring state tracking between steps

### Metrics
- **Task success rate** (primary)
- **Reasoning trace coherence** (secondary — human evaluation of whether reasoning is correct even when action succeeds/fails)
- **Recovery rate** after perturbations (how often the robot correctly identifies and recovers from failures)

### Baselines
- Molmo2 (VLM only, no action head) — as an ablation: can do visual QA but no actions
- OpenVLA (no reasoning) — direct action prediction without intermediate reasoning
- Standard behavioral cloning on the same data (no VLM backbone)

**Key results:**
- MolmoAct2 outperforms standard VLA approaches (no reasoning) particularly on **ambiguous tasks** and **novel object configurations** — exactly the settings where reasoning matters
- On simple unambiguous tasks, performance is comparable to OpenVLA-style direct action prediction
- Reasoning traces are judged coherent by human evaluators in >85% of cases

## Pros

- **Explicit reasoning** — human-interpretable intermediate steps; debuggable failures
- **Better OOD generalization** — reasoning enables compositional generalization to novel scenarios
- **Ambiguity resolution** — reasoning handles multi-object, multi-step ambiguities that confuse direct-action VLAs
- **Molmo2 foundation** — benefits from Molmo2's strong video understanding and visual grounding
- **Open weights** — accessible for fine-tuning by the research community

## Cons

- **Slower inference** — generating reasoning text before actions adds latency; not suitable for high-frequency control (>25Hz)
- **Reasoning quality depends on training data** — if demonstrations don't include correct reasoning traces, the model may learn plausible-sounding but incorrect reasoning
- **Action-reasoning consistency** — the model can generate correct reasoning but still produce incorrect actions (or vice versa); consistency is not guaranteed
- **Training data collection** — annotating demonstrations with reasoning traces is expensive; distillation from larger models introduces noise
- **Real-world deployment gap** — despite real-world focus, performance on truly out-of-distribution real environments requires ongoing evaluation
